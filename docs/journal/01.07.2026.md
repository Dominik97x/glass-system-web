# Future pricing structure

Prepared modular pricing architecture.

Pricing data will be separated into:

- construction
- walls
- roof
- ZIP
- awning
- LED
- accessories
- installation

This structure mirrors the EcoGardens pricing document while remaining independent from the pricing engine.

---

# Long-term architectural decision

Pricing Engine will never access pricing tables directly.

Instead it will communicate through a PriceRepository abstraction.

This allows future migration from static TypeScript files to PostgreSQL without changing business logic.

Target architecture:

Pricing Engine

↓

Price Repository

↓

File Repository (MVP)

↓

Database Repository (future)
